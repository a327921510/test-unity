# 需求完善与闭环检查 v1

### 1.1 地形与地图相关（已定版）
- 统一以 `地形与地图模块_设计_v1.md` 为唯一规则来源（地形枚举、可通行、补给可达、视野、高山阻挡等）。

### 1.2 城市与资源相关（已定版）

### 1.3 补给与断粮相关（已定版）
- 统一以 `补给与后勤模块_设计_v1.md` 为唯一规则来源（运输兵补给限制、资源类型、断粮惩罚、战法点联动等）。

### 1.4 跨模块约束（引用）
- 地形可达由 `地形与地图模块_设计_v1.md` 输出，补给执行由 `补给与后勤模块_设计_v1.md` 处理。
- 城市资源与人口治安口径由 `城市发展模块_设计_v1.md` 统一定义，其他模块仅消费输出。
- 全局占格固定规则：`1格1部队`（不可叠堆）+ `城市固定7格占据（中心1格+周围6格）` + **`城市所占7格对部队移动不可通行`** + **`同7格阻挡视线`** + **`城格内永无部队单位`** + **城间核心格距离≥50** + **非己军部队6邻格可进入但进入后行动力归0（ZOC）**（详见 `地形与地图模块_设计_v1.md` 与 `系统总览_模块关系图_v1.md` 5.9 节）。
- 全局行动经济规则：**移后可攻**；单回合内在未执行操作且有行动力时可持续移动（可前后反复）；单回合仅 1 次操作（普通进攻/战法/策略/补给），执行后行动力归 0 并进入移动完毕状态。
- 移动失败与优先级规则：当 `remainingMovePoint < moveCost` 时返回 `INSUFFICIENT_MOVE_POINT`；占格判定优先于 ZOC 判定（占格失败不触发 ZOC）；若行动力不足以进入任意一格，则可移动范围为空但流程正常。
- 地形相关失败/终止口径统一使用原因码规范（`地形与地图模块_设计_v1.md` 第 3.3 节），用于程序判定、UI提示与回放日志。
- 全局回合状态规则：部队行为仅分`移动/操作`，部队状态仅分`未执行/已执行`；回合开始（轮到当前势力时自动触发）全部置为未执行，回合结束（手动触发）全部置为已执行；行动力仅影响移动范围，执行一次操作即置为已执行。
- 执行状态真值规则：统一以 `executionState` 作为唯一真值字段（`UNEXECUTED/EXECUTED`）；不维护独立 `actionUsed` 持久状态，避免状态冲突。
- 城市资源与辖区区划以 `城市发展模块_设计_v1.md` 为准；入城拆散触发与流程以 `兵种与部队编制模块_设计_v1.md` 为准。
- 入城拆解回库溢出规则：超过城市可接收上限的兵源/兵器/钱粮直接消失，不保留、不转存（以 `兵种与部队编制模块_设计_v1.md` 为准）。
- 本文档不再重复任何已在上述模块文档定版的规则明细。

## 2. 未闭环与待定项（必须补齐）

> 说明：以下条目均为必要元素，建议在进入 Unity 实作前先冻结为 v1 参数。

- `待定` 兵种重组是否受“城市行动次数”限制（无限制，城内兵源+兵器出城后变为兵种，战法、策略、补给、建设之后视作部队的完成行动）。
- `待定` 投石车编制口径：按“兵器数量”还是“独立单位数”计。（一个投石车 + 至少3000兵源，出一个投石车队，投石车的人数主要参与伤害和受到伤害计算）

### 2.2 补给与运输兵
