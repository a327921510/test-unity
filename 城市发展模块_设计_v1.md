# 城市发展模块设计 v1

## 1. 模块定位

`城市发展模块` 是一个复合模块，统一管理：
- 人口与治安子模块
- 城市经营子模块
- 资源经济子模块

目标是形成稳定的内循环：
`人口/治安状态 -> 城市经营结果 -> 资源结算 -> 反哺下一回合发展`

### 1.1 顶层变量（v1 默认值）
- `SECURITY_AUTO_DECAY_CYCLE_TURNS = 2`  
  治安每 2 回合触发一次自动下降。
- `SECURITY_AUTO_DECAY_BASE = 2`  
  自动下降基础值为 2 点。
- `SECURITY_DECAY_UNDER_INVASION = 2`  
  城市处于“被入侵”时，额外下降 2 点。
- `SECURITY_DECAY_UNDER_SIEGE = 4`  
  城市处于“围城”时，额外下降 4 点。
- `CITY_DURABILITY_MAX_SMALL = 3000`  
  小城耐久上限。
- `CITY_DURABILITY_MAX_MEDIUM = 5000`  
  中城耐久上限。
- `CITY_DURABILITY_MAX_LARGE = 8000`  
  大城耐久上限。
- `CITY_DURABILITY_MAX_METROPOLIS = 12000`  
  特大城耐久上限。
- `CITY_DURABILITY_RECOVERY_SMALL = 60`  
  小城每回合基础耐久恢复值。
- `CITY_DURABILITY_RECOVERY_MEDIUM = 90`  
  中城每回合基础耐久恢复值。
- `CITY_DURABILITY_RECOVERY_LARGE = 130`  
  大城每回合基础耐久恢复值。
- `CITY_DURABILITY_RECOVERY_METROPOLIS = 180`  
  特大城每回合基础耐久恢复值。
- `CONSCRIPTION_GAIN_RATE_PER_TURN = 0.001`  
  兵役每回合自然增长比例（按人口的千分之一）。
- `CONSCRIPTION_TO_TROOP_SOURCE_RATE = 1`  
  兵役转兵源系数（1 点兵役可转化 1 点兵源）。
- `CONSCRIPTION_TO_TROOP_SOURCE_ROUNDING = floor`  
  兵役转兵源取整方式（向下取整）。
- `CONSCRIPTION_CONSUMED_SMALL = 1000`  
  小城每次执行招募时默认消耗兵役。
- `CONSCRIPTION_CONSUMED_MEDIUM = 1500`  
  中城每次执行招募时默认消耗兵役。
- `CONSCRIPTION_CONSUMED_LARGE = 2000`  
  大城每次执行招募时默认消耗兵役。
- `CONSCRIPTION_CONSUMED_METROPOLIS = 2500`  
  特大城每次执行招募时默认消耗兵役。
- `CITY_COEFF_SMALL = 0.8`  
  小城六类系数统一值（人口增长/钱/粮/兵源/兵器/投石车）。
- `CITY_COEFF_MEDIUM = 1.0`  
  中城六类系数统一值（人口增长/钱/粮/兵源/兵器/投石车）。
- `CITY_COEFF_LARGE = 1.2`  
  大城六类系数统一值（人口增长/钱/粮/兵源/兵器/投石车）。
- `CITY_COEFF_METROPOLIS = 1.5`  
  特大城六类系数统一值（人口增长/钱/粮/兵源/兵器/投石车）。
- `POP_GROWTH_RATE_SMALL = 0.002`  
  小城和平状态人口基础增长率（每回合）。
- `POP_GROWTH_RATE_MEDIUM = 0.003`  
  中城和平状态人口基础增长率（每回合）。
- `POP_GROWTH_RATE_LARGE = 0.004`  
  大城和平状态人口基础增长率（每回合）。
- `POP_GROWTH_RATE_METROPOLIS = 0.005`  
  特大城和平状态人口基础增长率（每回合）。
- `SECURITY_GROWTH_THRESHOLD_SMALL = 45`  
  小城人口增长治安阈值（达到后才可增长）。
- `SECURITY_GROWTH_THRESHOLD_MEDIUM = 55`  
  中城人口增长治安阈值（达到后才可增长）。
- `SECURITY_GROWTH_THRESHOLD_LARGE = 65`  
  大城人口增长治安阈值（达到后才可增长）。
- `SECURITY_GROWTH_THRESHOLD_METROPOLIS = 75`  
  特大城人口增长治安阈值（达到后才可增长）。
- `POP_INVASION_GROWTH_FACTOR = 0.4`  
  被入侵时人口增长系数（相对和平放缓）。
- `POP_SIEGE_LOSS_RATE = 0.002`  
  围城状态人口下降率（每回合）。
- `LOW_SECURITY_OUTPUT_PENALTY_FACTOR = 0.5`  
  治安低于 50 时的统一产出惩罚系数（所有产出降为 50%）。
- `WEAPON_PRODUCTION_PER_ACTION = 200`  
  每次手动兵器生产默认产量（按所选兵器类型入库）。
- `CATAPULT_PRODUCTION_PROGRESS_PER_ACTION = 0.25`  
  每次手动投石车生产默认进度（累计 1.0 产出 1 台）。

## 2. 模块边界（做什么 / 不做什么）

## 2.1 负责内容
- 管理城市人口、治安、战乱状态。
- 管理城市等级、发展行为、耐久恢复。
- 管理钱、粮、兵器、兵源、投石车生产进度与资源结算。
- 对外提供“可编制能力”和“资源可用量”。

### 本轮新增共识（已写入）
- 城市等级为 4 级：小城、中城、大城、特大城。
- 人口达到阈值后自然升级城市等级（无需额外升级操作）。
- 城市状态枚举：和平、被入侵、围城。
- 治安是 0-100 的城市属性；可手动提升；会随回合自然下降；被入侵/围城时下降加剧。
- 兵器生产、投石车生产为手动执行操作。
- 兵器生产需手动选择类型：枪、戟盾、马、弓。
- 城市新增“兵役”属性：兵役按人口每回合自然增长，招募兵源时由兵役转化而来。

## 2.2 不负责内容
- 不处理具体战斗计算（伤害、命中、克制）。
- 不处理部队移动与路径。
- 不处理战法释放判定细节。
- 不处理 AI 决策（只提供状态数据给 AI 使用）。

## 3. 对外输入输出定义（模块级）

## 3.1 输入（Input）

### A. 回合触发输入
- `turnIndex`: 当前回合序号（按月）。
- `phase`: 结算阶段标记（回合开始/回合结束）。

### B. 战争影响输入（来自战斗攻城模块）
- `cityUnderAttack`: 城市本回合是否被攻击。
- `battleOccurredInRegion`: 城市辖区是否发生战斗。
- `cityOwnerChanged`: 城市是否易主。
- `cityDurabilityDelta`: 城市耐久变化量。
- `cityTroopSourceDelta`: 城市兵源库存变化量。

### C. 内政行为输入（来自玩家/系统操作）
- `devActions`: 本回合发展操作列表（农、商、训练、修复等）。
- `rewardActions`: 将领奖励行为（忠诚提升消耗）。
- `constructionActions`: 建设行为（建筑前置、资源消耗）。
- `weaponProductionActions`: 兵器手动生产操作（需带兵器类型：枪/戟盾/马/弓）。
- `catapultProductionActions`: 投石车手动生产操作。
- `recruitActions`: 兵源招募操作（兵役转兵源）。

### D. 参数与规则输入（来自数值治理模块）
- `formulaSetVersion`: 当前公式版本号。
- `hardLimits`: 硬阈值（上限、下限、强制截断）。
- `softLimits`: 软阈值（衰减、边际收益递减）。

## 3.2 输出（Output）

### A. 城市状态输出
- `cityLevel`: 城市等级（小/中/大/特大）。
- `cityDurability`: 当前耐久。
- `cityPopulation`: 当前人口。
- `citySecurity`: 当前治安。
- `cityWarState`: 当前战乱状态（和平/被入侵/围城）。

### B. 资源结算输出
- `moneyDelta`
- `foodDelta`
- `weaponDelta`
- `conscriptionDelta`
- `troopSourceDelta`
- `catapultProgressDelta`
- `resourceSnapshotAfterSettlement`

### C. 对外能力输出（供其他模块使用）
- `maxRecruitableTroopSource`: 可招募兵源上限。
- `maxComposableForces`: 当前可编制能力（按资源计算）。
- `supplyCapacityHint`: 后勤建议值（供补给模块/AI参考）。

### D. 事件与告警输出
- `alerts`: 治安低于阈值、资源不足、人口增长停滞等告警。
- `logEntries`: 本回合城市发展结算日志。

## 4. 子模块输入输出（分层）

## 4.1 人口与治安子模块

**输入**
- 城市等级
- 当前治安
- 当前战乱状态
- 是否被攻击/辖区战斗
- 特定内政行为（治安整顿等）

**输出**
- `populationDelta`
- `securityDelta`
- `warStateDelta`
- `penaltyFlags`（如低治安惩罚触发）

### 城市等级与人口阈值（已确认）
- 小城：人口 < 50 万
- 中城：50 万 <= 人口 < 100 万
- 大城：100 万 <= 人口 < 150 万
- 特大城：人口 >= 150 万

### 城市状态定义（已确认）
- 和平：未被敌军直接影响的常规状态。
- 被入侵：回合开始判定时，城市辖区内存在其他势力部队。
- 围城：回合开始判定时，城市周围 1 格存在其他势力部队。

### 治安变化规则（已确认方向）
- 手动提升：通过内政行为提升治安。
- 自动下降：按照 `SECURITY_AUTO_DECAY_CYCLE_TURNS` 周期触发，基础下降 `SECURITY_AUTO_DECAY_BASE`。
- 战争加剧：被入侵、围城会在自动下降基础上附加额外下降。
- 计算建议：若同回合同时满足“被入侵”和“围城”，以“围城”规则优先计算，避免重复叠加。

### 已确认补充
- 特大城人口无上限。

---

## 4.2 城市经营子模块

**输入**
- 人口与治安子模块输出结果
- 当前城市等级与耐久
- 发展/建设/修复行为
- 可用资源

**输出**
- `cityLevelDelta`
- `durabilityRecoveryBase`
- `productionCapacityModifier`

### 城市等级系数（已确认方向）
每个城市等级都应拥有独立系数：
- 人口增长系数
- 钱获取系数
- 粮获取系数
- 兵源招募系数
- 兵器生产系数
- 投石车生产系数

### 四级城市六类系数（已确认默认值）
- 小城：`CITY_COEFF_SMALL = 0.8`
- 中城：`CITY_COEFF_MEDIUM = 1.0`
- 大城：`CITY_COEFF_LARGE = 1.2`
- 特大城：`CITY_COEFF_METROPOLIS = 1.5`
- 说明：当前六类系数统一使用同一档位值，后续若要细化可拆分为独立系数字段。

### 城市耐久默认值（已新增）
- 小城：耐久上限 `CITY_DURABILITY_MAX_SMALL`，每回合恢复 `CITY_DURABILITY_RECOVERY_SMALL`。
- 中城：耐久上限 `CITY_DURABILITY_MAX_MEDIUM`，每回合恢复 `CITY_DURABILITY_RECOVERY_MEDIUM`。
- 大城：耐久上限 `CITY_DURABILITY_MAX_LARGE`，每回合恢复 `CITY_DURABILITY_RECOVERY_LARGE`。
- 特大城：耐久上限 `CITY_DURABILITY_MAX_METROPOLIS`，每回合恢复 `CITY_DURABILITY_RECOVERY_METROPOLIS`。

### 已确认补充（本轮）
- 城市自然升级：回合开始判断人口，达到标准自动升级。
- 降级规则：人口下降不触发城市降级。

### 待确认占位（数值参数）
- 暂无（本段待确认项已清空）。

## 4.3 资源经济子模块

**输入**
- 城市经营子模块产能修正
- 人口与治安惩罚标记
- 资源池当前值
- 投石车生产进度参数
- 手动生产操作（兵器、投石车）
- 招募操作（兵役转兵源）

**输出**
- 五类资源净变化值（钱、粮、兵器、兵源、投石车生产进度）
- `conscriptionDelta`（兵役净变化值）
- 资源不足告警与处理结果
- 可编制能力估算值

### 兵役系统（已新增）
- 定义：兵役是独立城市资源，用于招募兵源。
- 自然增长：每回合按人口自然增长。  
  公式：`conscriptionGain = floor(cityPopulation * CONSCRIPTION_GAIN_RATE_PER_TURN)`
- 招募转换：执行招募操作时，兵役按系数转换为兵源。  
  公式：`troopSourceFromConscription = floor(conscriptionConsumed * CONSCRIPTION_TO_TROOP_SOURCE_RATE)`
- 下限规则：当兵役为 0 时，执行招募操作结果为 0。
- 默认消耗：`conscriptionConsumed` 根据城市等级取默认值：  
  小城 `CONSCRIPTION_CONSUMED_SMALL`，中城 `CONSCRIPTION_CONSUMED_MEDIUM`，大城 `CONSCRIPTION_CONSUMED_LARGE`，特大城 `CONSCRIPTION_CONSUMED_METROPOLIS`。

### 兵器与投石车生产（已新增）
- 兵器生产：必须手动触发，且必须选择兵器类型（枪、戟盾、马、弓）。
- 投石车生产：必须手动触发，不参与自动月度生产。
- 默认产量：每次兵器生产 `WEAPON_PRODUCTION_PER_ACTION = 200`；每次投石车生产累计 `CATAPULT_PRODUCTION_PROGRESS_PER_ACTION = 0.25`，累计到 1.0 时入库 1 台投石车。
- 说明：v1 不启用投石车维护成本，自动结算不扣维护。

## 5. 内部逻辑流程（推荐顺序）

每回合结算建议固定顺序，避免循环依赖：

1. **接收外部状态变更**  
   应用战争结果（被攻击、易主、耐久变化、兵源变化）。

2. **执行人口与治安结算**  
   计算人口增长、治安变动、战乱影响与惩罚标记。

3. **执行城市经营结算**  
   应用发展行为与修复行为，计算城市能力变化。

4. **执行资源经济结算**  
   先结算兵役自然增长，再执行手动生产/手动招募，再计算其他资源变化，最后执行下限规则。

5. **生成对外输出与日志**  
   生成城市状态快照、资源快照、告警信息，输出给编制/补给/AI。

## 6. 数据一致性规则（建议）

- 所有数值先累计，再统一做上下限截断（避免中途截断误差）。
- 结算日志必须可回放（至少记录每项 delta 与原因）。
- 城市发展模块只产出“状态与能力”，不直接改战斗模块内部状态。
- v1 无每月自然损耗；资源仅在“行为消耗”或“惩罚事件”发生时扣减。
- 当某项行为/惩罚触发扣减且资源不足时：本次行为/惩罚按“可扣多少扣多少”结算，不引入额外的跨资源扣减顺序。

1. 四级城市六类系数默认值（钱/粮/兵源/兵器/投石车/人口增长）：

3. 被入侵、围城附加治安下降值：  
   `SECURITY_DECAY_UNDER_INVASION = 2`，`SECURITY_DECAY_UNDER_SIEGE = 4`。
4. 城市耐久基础恢复值（分城级）与耐久上限：  
   `CITY_DURABILITY_MAX_*` 与 `CITY_DURABILITY_RECOVERY_*` 按 1.1 与 4.2 节执行。
5. 钱、粮、兵器、兵源月度产出公式默认模板：  
   按第 10 节公式执行（含 `BASE_*_PER_POP`、`securityFactor`、`warPenalty`）。
6. 四级城市招募兵役默认消耗值：  
   `1000/1500/2000/2500`（小/中/大/特大）。
7. 兵役转兵源规则：  
   `CONSCRIPTION_TO_TROOP_SOURCE_RATE = 1`，且当前默认不受城市等级/治安修正。
8. 人口增长规则：  
   城市越大基础增长率越高，且增长所需治安阈值越高；被入侵时增长放缓；围城时人口按固定比例下降。
9. 治安低于 50 的惩罚规则：  
   统一对所有产出施加 `LOW_SECURITY_OUTPUT_PENALTY_FACTOR = 0.5`。
10. 耐久“花钱加速恢复”：  
   v1 不做（仅保留基础耐久恢复）。
11. 投石车维护成本：  
   v1 不做（默认不扣维护成本）。
12. 手动生产默认值：  
   兵器每次 `200`；投石车每次 `0.25` 进度（4 回合 1 台）。

## 10. 月度产出公式模板（v1）

## 10.1 变量定义

- `P`: 当前城市人口
- `SEC`: 当前城市治安（0-100）
- `cityCoeff`: 城市等级系数（小/中/大/特大 = 0.8/1.0/1.2/1.5）
- `warPenalty`: 战乱惩罚系数（和平=1.0，被入侵=0.9，围城=0.01，默认值可调）
- `securityFactor`: 治安修正系数

建议使用：
- `securityFactor = 0.5 + (SEC / 200)`  
  说明：SEC=100 时为 1.0，SEC=50 时为 0.75，SEC=0 时为 0.5。

基础产出常量（可调）：
- `BASE_MONEY_PER_POP = 0.12`
- `BASE_FOOD_PER_POP = 0.18`
- `BASE_WEAPON_PER_POP = 0.02`
- `BASE_TROOP_SOURCE_PER_POP = 0.015`

---

## 10.2 资源月度产出公式

统一模板：
- `resourceOutput = floor(P * BASE_RESOURCE_PER_POP * cityCoeff * securityFactor * warPenalty)`

展开后：
- `moneyOutput = floor(P * BASE_MONEY_PER_POP * cityCoeff * securityFactor * warPenalty)`
- `foodOutput = floor(P * BASE_FOOD_PER_POP * cityCoeff * securityFactor * warPenalty)`
- `weaponOutput = floor(P * BASE_WEAPON_PER_POP * cityCoeff * securityFactor * warPenalty)`
- `troopSourceOutput = floor(P * BASE_TROOP_SOURCE_PER_POP * cityCoeff * securityFactor * warPenalty)`
- 当 `SEC < 50` 时，所有产出统一乘以 `LOW_SECURITY_OUTPUT_PENALTY_FACTOR = 0.5`。

兵役月度增长：
- `conscriptionGain = floor(P * CONSCRIPTION_GAIN_RATE_PER_TURN)`

招募转换（手动动作）：
- `troopSourceFromConscription = floor(conscriptionConsumed * CONSCRIPTION_TO_TROOP_SOURCE_RATE)`

## 10.3 人口增长公式（v1 冻结）

按城市等级定义基础增长率与治安阈值：
- 小城：`baseGrowthRate = POP_GROWTH_RATE_SMALL`，`securityThreshold = SECURITY_GROWTH_THRESHOLD_SMALL`
- 中城：`baseGrowthRate = POP_GROWTH_RATE_MEDIUM`，`securityThreshold = SECURITY_GROWTH_THRESHOLD_MEDIUM`
- 大城：`baseGrowthRate = POP_GROWTH_RATE_LARGE`，`securityThreshold = SECURITY_GROWTH_THRESHOLD_LARGE`
- 特大城：`baseGrowthRate = POP_GROWTH_RATE_METROPOLIS`，`securityThreshold = SECURITY_GROWTH_THRESHOLD_METROPOLIS`

和平 / 被入侵状态下：
- 若 `SEC < securityThreshold`，则 `populationDelta = 0`
- 若 `SEC >= securityThreshold`，则  
  `populationDelta = floor(P * baseGrowthRate * securityBonus * warStateFactor)`
- `securityBonus = 1 + (SEC - securityThreshold) / 100`
- `warStateFactor`：和平 `= 1.0`，被入侵 `= POP_INVASION_GROWTH_FACTOR`

围城状态下（优先级最高）：
- `populationDelta = -floor(P * POP_SIEGE_LOSS_RATE)`

说明：
- 城市越大，`baseGrowthRate` 越高（增长速度越快）。
- 城市越大，`securityThreshold` 越高（达到阈值才可增长）。
- 被入侵会放缓增长，围城会导致人口下降。

## 10.4 计算示例 A（中城，和平，治安 80）

已知：
- `P = 800,000`
- `cityCoeff = 1.0`（中城）
- `SEC = 80` -> `securityFactor = 0.5 + 80/200 = 0.9`
- `warPenalty = 1.0`（和平）

计算：
- `moneyOutput = floor(800000 * 0.12 * 1.0 * 0.9 * 1.0) = floor(86400) = 86400`
- `foodOutput = floor(800000 * 0.18 * 1.0 * 0.9 * 1.0) = floor(129600) = 129600`
- `weaponOutput = floor(800000 * 0.02 * 1.0 * 0.9 * 1.0) = floor(14400) = 14400`
- `troopSourceOutput = floor(800000 * 0.015 * 1.0 * 0.9 * 1.0) = floor(10800) = 10800`
- `conscriptionGain = floor(800000 * 0.001) = 800`

若本回合执行一次招募（中城默认）：
- `conscriptionConsumed = CONSCRIPTION_CONSUMED_MEDIUM = 1500`
- `troopSourceFromConscription = floor(1500 * 1) = 1500`

则本回合兵源总新增（不含战损等其他项）：
- `10800 + 1500 = 12300`

## 10.6 使用建议

- 首版先固定这套模板跑 20-30 回合观察曲线。
- 若资源膨胀太快，优先下调 `BASE_*_PER_POP`；不要先改城级系数。
- 若弱城无存在感，微调 `CITY_COEFF_SMALL` 与 `CITY_COEFF_MEDIUM`。
- 围城惩罚当前采用极强抑制：`warPenalty = 0.01`，若需要缓和可回调至 0.1-0.3 区间。
