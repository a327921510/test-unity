# 战斗与攻城模块设计 v1

## 1. 模块定位

`战斗与攻城模块` 负责承接部队实体与空间判定结果，完成野战与攻城的命中、伤害、战损、破城与占领结算，并将结果回写到城市发展与部队运行态。

目标是形成稳定链路：
`部队状态 + 地形判定 + 后勤状态 -> 战斗/攻城结算 -> 城市/部队状态变更 -> 回流经营循环`

---

## 2. 模块边界（做什么 / 不做什么）

## 2.1 负责内容
- 普通攻击、战法攻击、反击的命中与伤害结算。
- 兵种克制、地形战斗修正、断粮状态修正的综合应用。
- 战损结果生成（兵源/兵器损失）与战场单位存亡判定。
- 攻城行为结算（城市耐久扣减、城市兵源扣减、破城判定、占领判定）。
- 战斗结果日志输出（供 UI 展示与回放）。

## 2.2 不负责内容
- 不负责路径可达、视野遮挡、占格冲突、ZOC 进入终止判定（由 `地形与地图模块_设计_v1.md` 负责）。
- 不负责部队编制、入城拆解、部队上限规则（由 `兵种与部队编制模块_设计_v1.md` 负责）。
- 不负责补给执行与补给距离判定（由 `补给与后勤模块_设计_v1.md` 负责）。
- 不负责城市月度产出、人口治安、耐久自然恢复（由 `城市发展模块_设计_v1.md` 负责）。

## 3. 依赖输入与模块输出

## 3.1 输入（Input）

### A. 部队快照输入（来自编制模块）
- `attackerUnitSnapshot`
- `defenderUnitSnapshot`
- `armsType / troopSource / weaponAmount / catapultAmount`
- `finalStatSnapshot`（`finalAtk / finalDef / finalMove / finalInt / finalBuild`）
- `finalCombatDetailSnapshot`（`finalMeleeAtk / finalRangedAtk / finalMeleeDef / finalRangedDef`）

- `terrainTagAttacker / terrainTagDefender`
- `terrainCombatModifierSnapshot`

### C. 后勤与状态输入（来自后勤模块）
- `foodStatus`（正常/断粮）
- `supplyPenaltySnapshot`（攻击、防御、士气等修正）

### D. 城市目标输入（来自城市发展模块）
- `cityDurability`
- `cityDurabilityMax`
- `cityTroopSource`
- `cityOwnerFactionId`
- `cityWarState`

### E. 规则参数输入（来自数值治理模块）
- `hitFormulaRuleSet`
- `damageFormulaRuleSet`
- `armsCounterRuleTable`
- `siegeRuleTable`
- `woundedRuleTable`

## 3.2 输出（Output）

### A. 战斗输出
- `isBattleResolved`
- `hitResult`
- `damageResult`
- `counterAttackResult`
- `attackerLoss / defenderLoss`
- `attackerWoundedDelta / defenderWoundedDelta`
- `attackerRouted / defenderRouted`

### B. 攻城输出
- `cityDurabilityDelta`
- `cityTroopSourceDelta`
- `isCityBreached`
- `isCityCaptured`
- `cityOwnerChanged`
- `newOwnerFactionId`（仅当 `cityOwnerChanged = true` 时必填）

### C. 回写输出
- `unitDeltaList`（部队兵源、兵器、伤员、状态变化）
- `cityDeltaEvent`（供城市发展模块消费）：
  - `cityUnderAttack`
  - `battleOccurredInRegion`
  - `cityOwnerChanged`
  - `newOwnerFactionId`
  - `cityDurabilityDelta`
  - `cityTroopSourceDelta`
  - `cityWoundedTroopSourceDelta`
- `battleLogEntryList`

## 4. 核心流程（v1）

## 4.1 野战流程
1. 校验攻击前置：`executionState = UNEXECUTED` 且目标可达。
2. 读取攻击方与防守方最终属性快照。
3. 应用兵种克制、地形修正、后勤状态修正。
4. 执行命中判定；未命中则进入日志并结束本次伤害。
5. 命中后执行伤害计算并扣减目标兵源/兵器。
6. 按伤员转换系数计算本次新增伤员并写入目标部队：
   - `newWounded = floor(lossTroopSource * WOUNDED_FROM_LOSS_COEFF)`（v1 默认 `WOUNDED_FROM_LOSS_COEFF = 0.4`）。
   - 伤员仅由战斗战损产生，出征初始化为 0。
7. 若目标可反击且仍存活，执行反击判定与反击伤害（反击造成的战损同样生成伤员）。
8. 生成战损结果，若 `troopSource <= 0` 则标记溃败并移除实体；部队被移除时其 `woundedTroopSource` 一并清空（不回收）。
9. 攻击方本次操作结束：置 `EXECUTED` 且行动力归 0（与全局规则一致）。

## 4.2 攻城流程
1. 校验攻城前置：目标为敌方城市，部队位于合法攻城邻格（由地图模块保证）。
2. 判断攻击类型：普通攻城（常规部队）/器械攻城（投石车）。
3. 计算本次对城市耐久伤害，并写入 `cityDurabilityDelta`。
4. 若规则允许同步打击城内兵源，则计算并写入 `cityTroopSourceDelta`。
5. 先应用本次增量，得到城市当前值：
   - `cityDurabilityAfter = max(0, cityDurability + cityDurabilityDelta)`
   - `cityTroopSourceAfter = max(0, cityTroopSource + cityTroopSourceDelta)`
6. 执行破城判定：
   - 条件 A：`cityDurabilityAfter <= 0`
   - 条件 B：`cityTroopSourceAfter <= 0 and cityDurabilityAfter > 0`（当前口径：满足即破城）
7. 破城后执行占领判定与归属变更，回写 `cityOwnerChanged` 与 `newOwnerFactionId`。
8. 输出战争结果事件（含 `cityUnderAttack` 与 `battleOccurredInRegion`），供城市发展模块更新战乱状态与后续结算。

## 5. 结算口径（v1）

## 5.1 命中公式
- v1 冻结模板：
  - `hitRate = clamp(BASE_HIT + atkHitBonus - defDodgeBonus + terrainHitModifier + statusHitModifier, HIT_MIN, HIT_MAX)`
  - `isHit = random(0, 1) <= hitRate`
- 说明：`BASE_HIT / HIT_MIN / HIT_MAX` 的具体值由数值治理模块参数表配置。

## 5.2 伤害公式
- v1 冻结模板：
  - `rawDamage = max(1, attackValue - defenseValue)`
  - `finalDamage = floor(rawDamage * armsCounterCoeff * terrainDamageCoeff * statusDamageCoeff * skillCoeff)`
- 说明：乘区规则按数值治理模块统一定义，不在本模块内分散维护。

## 5.3 攻城伤害公式
- v1 冻结模板：
  - `siegeDamage = floor(baseSiegePower * siegeTypeCoeff * terrainSiegeCoeff * statusSiegeCoeff)`
  - `cityDurabilityAfter = max(0, cityDurability - siegeDamage)`

## 5.4 战损、伤员与移除
- 部队溃败条件：`troopSource <= 0`。
- 当部队溃败时，地图单位实体移除，并输出战损日志。
- 伤员由战损产生，转换公式：
  - `woundedDelta = floor(lossTroopSource * WOUNDED_FROM_LOSS_COEFF)`。
- v1 冻结：`WOUNDED_FROM_LOSS_COEFF = 0.4`。
- 部队被消灭时，伤员不回收，随部队实体一并消失。

## 6. 兵种与攻城特例（v1 口径）

## 6.1 兵种克制关系
- 以冻结口径执行：`骑兵 -> 戟盾兵 -> 枪兵 -> 骑兵`。
- 弓兵与投石车走特殊规则，不直接纳入该闭环克制链。

## 6.2 弓兵规则
- 弓兵按远程维度结算（读取 `finalRangedAtk` / `finalRangedDef`）。
- 弓兵 `1.2` 倍率保留为参数位：`ARCHER_SPECIAL_COEFF`（由数值治理冻结，当前建议定义为“最终伤害乘区”）。
- 弓兵在特定遮挡地形的可攻击性由地图模块先行判定，本模块仅消费可攻击结果。

## 6.3 投石车规则
- 投石车对城默认使用 `siegeTypeCoeff` 增益结算。
- 投石车“普通对军伤害倍率”保留参数位：`CATAPULT_VS_UNIT_COEFF`（由数值治理冻结）。

## 6.4 围城与破城
- 围城状态识别由城市发展模块在回合开始判定（周边 1 格敌军）。
- 本模块仅依据输入状态对攻城伤害或士气参数施加修正（如存在）。
- 破城条件采用双阈值口径（耐久归零或城内兵源归零）。

---

## 7. 跨模块接口对齐

## 7.1 对地图模块
- 只消费 `可攻击/不可攻击`、攻击距离类型、地形修正标签。
- 不二次计算可达性，避免重复判定造成冲突。

## 7.2 对后勤模块
- 只消费断粮状态修正结果，不在战斗层重算粮草逻辑。
- 断粮导致的战斗能力下降必须在战斗日志中显式记录来源。

## 7.3 对城市发展模块
- 输出 `cityUnderAttack / battleOccurredInRegion / cityDurabilityDelta / cityTroopSourceDelta / cityWoundedTroopSourceDelta / cityOwnerChanged / newOwnerFactionId`。
- 城市耐久自然恢复、战乱持续惩罚在城市模块回合结算中执行。

## 7.4 对编制模块
- 输出部队兵源与兵器变化结果。
- 部队死亡后只返回“溃败移除”事件，不直接处理入城与回库逻辑。

## 8. 参数冻结清单（v1）

## 8.1 已冻结
1. 克制闭环：`骑兵 -> 戟盾兵 -> 枪兵 -> 骑兵`。
2. 全局行动经济：单回合仅 1 次操作，执行操作后 `executionState = EXECUTED` 且行动力归 0。
3. 破城补充判定：`cityTroopSource = 0 and cityDurability > 0` 时可判定破城。
4. 破城判定时点：以应用本次 `cityDurabilityDelta / cityTroopSourceDelta` 后的 `After` 值为准。
5. 城市易主输出契约：`cityOwnerChanged = true` 时必须提供 `newOwnerFactionId`。
6. 城市战争事件输出契约：必须输出 `cityUnderAttack` 与 `battleOccurredInRegion`。
7. 命中/伤害/攻城公式模板在 v1 固定为第 5 章口径，仅参数值可配。
8. 伤员机制输出契约：战斗输出需包含 `attackerWoundedDelta / defenderWoundedDelta`。
9. 伤员来源冻结：仅战斗战损可生成伤员；出征初始化为 0。
10. 部队灭亡伤员去向冻结：部队被移除时其伤员直接消失，不回收。

## 8.2 已冻结补充（按 `规则冻结_v1.md`）
1. `ARCHER_SPECIAL_COEFF_APPLY_STAGE = FINAL_DAMAGE`（弓兵系数作用于最终伤害乘区）。
2. `ARCHER_SPECIAL_COEFF = 1.20`。
3. `CATAPULT_VS_UNIT_COEFF = 0.70`（v1-default，可调）。
4. 战法点消耗按类别固定值执行（见 `规则冻结_v1.md`）。
5. 战法失败条件默认启用地形/士气/兵力限制，天气在 MVP 默认关闭（见 `规则冻结_v1.md`）。

## 9. 验收检查点（MVP）

- 可完成一次完整野战：命中 -> 伤害 -> 反击 -> 战损 -> 溃败移除。
- 可完成一次完整攻城：攻城伤害 -> 城市耐久变化 -> 破城 -> 占领变更。
- 战斗结果能够回写城市发展与部队状态，并在下一回合被正确消费。
- 同一条战斗事件可在日志中追溯到“兵种/地形/后勤/战法”各修正来源。
