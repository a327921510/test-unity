# Unity 开发执行清单 v1

## 1. 使用说明

本清单用于把现有文档直接转成工程任务。  
规则口径统一引用 `规则冻结_v1.md`，实现时不再重复讨论规则定义。

---

## 2. 第一阶段（先跑通最小闭环）

目标：完成“经营 -> 编制 -> 移动战斗 -> 攻城占领 -> 回写经营”的可运行链路。

### 2.1 数据与配置底座
- 建立参数资产：`GlobalRulesConfig`、`TerrainRulesConfig`、`CombatRulesConfig`、`LogisticsRulesConfig`、`CityRulesConfig`、`TacticRulesConfig`、`LoyaltyRulesConfig`。
- 所有参数字段名与 `规则冻结_v1.md` 一致。
- 增加全局版本字段：`RULE_FREEZE_VERSION = v1`。

### 2.2 地图与回合
- 六边形网格数据结构与环距离查询。
- 回合循环：势力轮转、回合开始/结束状态重置。
- 占格与城市 7 格阻挡规则。
- 敌军邻格压制（ZOC）与移动终止原因码。

### 2.3 部队与编制
- 城市资源扣减 -> 部队创建。
- 部队字段：兵源、兵器、钱粮、状态、行动力。
- 入城拆解：邻格触发、回库、溢出丢弃。

### 2.4 战斗与攻城
- 命中/伤害/反击公式模板接入。
- 弓兵与投石车特殊参数位接入（读配置，不写死）。
- 攻城耐久变化、破城判定、易主事件输出。

### 2.5 城市与后勤
- 月度产出、兵役增长、招募转换。
- 伤员池与自动恢复。
- 运输兵补给流程与断粮惩罚。

---

## 3. 第二阶段（可玩验证）

目标：让 1 张测试地图从开局跑到占领胜利条件。

- 做 1 张最小测试图（2 势力、3-5 城）。
- 接入基础 UI：行动范围、攻击预览、资源变化、失败原因提示。
- 接入战报日志：至少记录地形/克制/后勤修正来源。
- 做最小 AI：可移动、可攻击、可攻城。

---

## 4. 第三阶段（MVP 扩展）

- 扩到 3 张可通关地图。
- 扩到 MVP 兵种与技能数量。
- 补事件与外交最小集合。
- 开始数值调优，保持公式不变只调参数。

---

## 5. 验收清单（必须全过）

- 能完成一轮完整野战：命中 -> 伤害 -> 反击 -> 溃败移除。
- 能完成一次完整攻城：扣耐久 -> 破城 -> 易主。
- 城市月结算正确影响编制与补给。
- 所有失败行为都能返回标准原因码并在 UI 展示。
- 所有计算日志可追溯 `RULE_FREEZE_VERSION`。

---

## 6. 今日可开工任务（建议顺序）

1. 建参数资产和版本字段。  
2. 实现回合与 `executionState` 状态机。  
3. 实现地图阻挡/ZOC/可移动范围。  
4. 接入最小战斗与攻城结算。  
5. 接入城市月结算与编制。  
6. 跑通第一条自动化规则测试。
