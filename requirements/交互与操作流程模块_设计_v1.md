# 交互与操作流程模块设计 v1

## 1. 模块定位

`交互与操作流程模块` 负责把玩家输入（鼠标/点击）转化为明确的游戏操作状态流，不负责底层规则结算。

目标是形成稳定链路：  
`输入事件 -> 选择态流转 -> 可操作反馈 -> 操作确认 -> 调用地图/部队/城市/战斗模块执行`

---

## 2. 模块边界（做什么 / 不做什么）

## 2.1 负责内容
- 定义鼠标边缘滚屏触发规则与相机移动口径。
- 定义对象点击优先级、选中态表现与取消选中规则。
- 定义“选中单位 -> 显示可达范围 -> 选择目的地 -> 逐格移动”的交互流程。
- 定义城市/部队选中后侧边操作面板的展示规则。

## 2.2 不负责内容
- 不负责移动可达范围本身的计算（由 `地形与地图模块_设计_v1.md` 提供）。
- 不负责部队属性、移动力、执行状态存储（由 `兵种与部队编制模块_设计_v1.md` 提供）。
- 不负责攻城/伤害/命中等战斗结算（由 `战斗与攻城模块_设计_v1.md` 提供）。
- 不负责城市资源变化与发展结算（由 `城市发展模块_设计_v1.md` 提供）。

---

## 3. 输入与状态机

## 3.1 输入事件（v1）
- `MouseMove(screenPos)`
- `LeftClick(worldPos)`
- `LeftPressStart(worldPos)`
- `LeftPressHold(worldPos, holdMs, dragDelta)`
- `LeftPressEnd(worldPos)`
- `RightClick(worldPos)`（v1 作为取消或返回上一级）
- `Esc`（强制取消当前选中并关闭侧栏）

## 3.2 全局交互状态
- `IDLE`：无选中对象。
- `CITY_SELECTED`：已选中城市，显示城市选中效果与城市操作面板。
- `UNIT_SELECTED`：已选中部队，显示部队选中效果、可达范围与部队操作面板。
- `UNIT_MOVE_TARGETING`：已确认“移动意图”，等待玩家点击目的格。
- `UNIT_MOVING`：部队沿路径逐格移动中，临时锁输入（仅允许相机移动与取消队列外输入缓存）。
- `CAMERA_DRAGGING`：玩家在空地左键长按并拖拽中，视角随鼠标平移。

状态转换简表：
- `IDLE -> CITY_SELECTED`：左键点中城市。
- `IDLE -> UNIT_SELECTED`：左键点中部队。
- `CITY_SELECTED -> IDLE`：点空地/右键/Esc。
- `UNIT_SELECTED -> IDLE`：点空地/右键/Esc。
- `UNIT_SELECTED -> UNIT_MOVE_TARGETING`：点击移动按钮或默认移动意图触发。
- `UNIT_MOVE_TARGETING -> UNIT_MOVING`：点击合法目的格。
- `UNIT_MOVE_TARGETING -> UNIT_SELECTED`：点击非法格/右键/Esc。
- `UNIT_MOVING -> UNIT_SELECTED`：移动结束且单位仍可执行（按外部执行状态决定）。
- `UNIT_MOVING -> IDLE`：移动结束后自动取消选中（v1 默认不自动保留选中）。
- `IDLE -> CAMERA_DRAGGING`：左键按在空地且达到长按+位移阈值。
- `CITY_SELECTED -> CAMERA_DRAGGING`：左键按在空地且达到长按+位移阈值。
- `UNIT_SELECTED -> CAMERA_DRAGGING`：左键按在空地且达到长按+位移阈值。
- `CAMERA_DRAGGING -> IDLE/CITY_SELECTED/UNIT_SELECTED`：左键释放后返回拖拽前主状态（不触发点击选中）。

---

## 4. 相机移动规则（边缘滚屏 + 长按拖拽）

## 4.1 触发条件
- 鼠标在游戏窗口内时才检测边缘滚屏。
- 当鼠标进入屏幕边缘阈值区域时，相机向对应方向移动。
- 支持八方向组合（上/下/左/右及四个对角）。

## 4.2 参数口径（v1 默认）
- `CAMERA_EDGE_SCROLL_ENABLED = true`
- `CAMERA_EDGE_THRESHOLD_PX = 24`
- `CAMERA_EDGE_SCROLL_SPEED = 18.0`（世界单位/秒）
- `CAMERA_SCROLL_USE_DELTA_TIME = true`
- `CAMERA_CLAMP_TO_MAP_BOUNDS = true`

## 4.3 边界约束
- 相机位置始终被夹在地图边界内，不允许滚出可玩区域。
- 地图边界数据由地图模块提供（中心点 + 可视半径或 AABB）。

## 4.4 空地左键长按拖拽视角（v1）
- 触发前提：左键按下位置必须是“无单位、非城市”的空地。
- 触发条件：同时满足“长按时长阈值”和“位移阈值”后进入 `CAMERA_DRAGGING`。
- 拖拽行为：视角按鼠标位移增量进行平移，持续到左键释放。
- 结束行为：左键释放退出 `CAMERA_DRAGGING`，不触发点击选中逻辑（防止拖拽误触）。
- 优先级：`CAMERA_DRAGGING` 期间暂停边缘滚屏，避免双重相机输入叠加。
- 默认参数：
  - `CAMERA_DRAG_ON_EMPTY_LEFT_HOLD_ENABLED = true`
  - `CAMERA_DRAG_LONG_PRESS_MS = 180`
  - `CAMERA_DRAG_START_THRESHOLD_PX = 8`

## 5. 选中与面板联动

## 5.1 点击优先级（同屏重叠时）
1. 部队
2. 城市
3. 地块空位

## 5.2 选中视觉反馈
- 被选中对象显示高亮描边或底部选中圈（城市与部队样式可不同）。
- 同一时刻仅允许一个主选中对象。
- 新对象选中时，旧对象选中效果立即清除。

## 5.3 侧边操作面板规则
- 选中城市：打开城市操作面板（发展、征募、编制入口等）。
- 选中部队：打开部队操作面板（移动、攻击、战法、补给入口等）。
- 取消选中：侧边面板关闭或切回默认信息面板。
- 面板只负责展示可用动作与触发请求，动作合法性由对应业务模块最终校验。

## 6. 部队移动交互流程（核心）

## 6.1 流程步骤
1. 玩家左键选中部队，进入 `UNIT_SELECTED`。
2. 系统向地图模块请求“当前可达范围”和“目的格合法性查询接口”。
3. 地图高亮可达范围（可达格）与不可达边界（可选）。
4. 玩家点击目的格：
   - 若非法：保持在 `UNIT_MOVE_TARGETING`，给出错误提示（不可达/被占用/越界）。
   - 若合法：生成路径并进入 `UNIT_MOVING`。
5. 部队按路径逐格移动（每格有离散步进表现）。
6. 移动结束后刷新单位状态、可执行动作与面板内容。

## 6.2 逐格移动表现口径
- 采用“路径节点逐格推进”而非瞬移。
- 每进入一格触发一次步进事件，供音效/动画/UI 使用。
- 默认参数：
  - `UNIT_MOVE_STEP_MODE = GRID_BY_GRID`
  - `UNIT_MOVE_STEP_TIME_SEC = 0.12`
  - `UNIT_MOVE_INTERPOLATION = LINEAR`

## 6.3 关键约束
- 不允许在 `UNIT_MOVING` 期间重复下发同单位移动命令。
- 目的格必须在当前查询得到的可达集合内，且通过最终合法性校验。
- 移动中若外部状态变化导致路径失效，立即中断移动并回滚到最近合法格（v1 可先实现为“停止在当前格并提示”）。

## 8. 验收用例（MVP）

- 鼠标移到窗口上边缘时，相机持续向上滚动；移出边缘阈值后停止。
- 鼠标贴住左上角时，相机沿左上方向斜向移动且不越出地图边界。
- 左键按在空地并长按后拖动鼠标，视角随拖拽平移；松开左键后停止拖拽。
- 左键按在单位或城市上并长按，不触发视角拖拽，保持对象交互优先级。
- 视角拖拽进行中，边缘滚屏不生效；拖拽结束后恢复边缘滚屏。
- 点击城市后出现城市选中效果与城市侧边面板；点空地后关闭。
- 点击部队后出现部队选中效果、可达范围高亮与部队侧边面板。
- 选中部队后点击不可达格，部队不移动并提示原因。
- 选中部队后点击可达格，部队按路径逐格移动到目标格。

---

## 9. 与既有模块的衔接说明

- 地图模块提供空间规则真值；本模块只消费结果并负责交互表达。
- 兵种与编制模块提供单位执行态与行动力真值；本模块不维护重复状态。
- 城市发展模块提供城市可操作动作清单；本模块仅负责显示与触发入口。
- 战斗与攻城模块负责动作结算；本模块只负责把“玩家意图”转成标准请求。
